# ⚡ Keep-Alive Monitor — مراقبة وإبقاء المواقع صاحية 24/7

## نظرة عامة
- **الاسم**: Keep-Alive Monitor
- **الهدف**: تضيف أي رابط موقع، والتطبيق يزوره تلقائياً (كل دقيقة أو حسب إعدادك) من السيرفر مباشرة — **حتى لو موبايلك مقفول تماماً** — عشان الموقع ميناماش أبداً.
- **مصمم للموبايل أولاً** 📱 بواجهة عربية كاملة (RTL)

## ✅ المميزات المكتملة
- ➕ إضافة/تعديل/حذف روابط بلا حدود
- ⏰ **كل رابط بإعداداته المستقلة**: فترة الزيارة (1 دقيقة → 30 دقيقة)، المهلة، إيقاف/تشغيل
- 🤖 **Cloudflare Cron Trigger** يزور الروابط المستحقة **كل دقيقة من السيرفر** (مش من موبايلك)
- 🧠 **زيارات ذكية (النسخة المطوّرة)**:
  - 🔁 **إعادة محاولة تلقائية (Smart Retries)** قبل اعتبار الرابط "واقع" — يمنع الإنذارات الكاذبة الناتجة عن تقطيع شبكة مؤقت
  - 🎭 **تدوير User-Agent** (Desktop + Mobile) لتقليل احتمال الحظر
  - 🔐 **وعي بتسجيل دخول جوجل**: لو الموقع حوّلك لصفحة تسجيل دخول (accounts.google.com) — يعتبره **صاحٍ ✅** (لأن السيرفر رد فعلاً) بدل ما يعتبره واقع
  - 🔎 **تحقق بكلمة مفتاحية (Keyword)** اختياري: يتأكد إن الصفحة حمّلت المحتوى الصح مش مجرد HTTP 200 فاضي
  - 📉 تتبّع عدّاد الأعطال المتتالية (consecutive fails) لكل رابط
- 📊 لوحة تحكم موبايل: حالة كل رابط (شغال ✅ / واقع ❌)، سرعة الاستجابة (min/avg/max)، نسبة Uptime، عدد الزيارات
- 📈 سجل آخر 30 زيارة لكل رابط + رسم بياني مصغر للسرعة
- 🖼️ **عرض الموقع الحي** داخل اللوحة (iframe) — الصفحة نفسها مفتوحة فعلياً
- ⚡ زر "افحص الآن" لفحص فوري يدوي
- 🔄 تحديث تلقائي للوحة كل 15 ثانية + تشغيل الفحوصات المستحقة كل دقيقة أثناء فتح اللوحة
- 🌱 **مُحمّل مسبقاً** بروابط تطبيق AI Studio الثلاثة (Preview / Assistant / Fullscreen) وجاهز للعمل فوراً

الملف `keepalive/links.json` **مُحمّل مسبقاً** بروابط تطبيق AI Studio الثلاثة، فبمجرد نسخ الـ workflow هيبدأ يزورها فوراً.

### طريقة الاستخدام اليومي (من موبايلك مباشرة):
1. افتح ملف `keepalive/links.json` في GitHub → اضغط ✏️ تعديل
2. ضيف رابط موقعك في قائمة `links` واحفظ (Commit) على فرع `main`
3. خلاص! GitHub هيزور موقعك كل 5 دقائق تلقائياً
4. شوف النتائج في `keepalive/status.json` أو في تبويب **Actions**
5. لتشغيل فحص فوري: تبويب **Actions** → **Keep-Alive Pinger 24/7** → **Run workflow**

### 🔑 لو الموقع محتاج تسجيل دخول جوجل إجباري:
- الزيارة من السيرفر بتحصل **بدون تسجيل دخول** — ودي كافية تماماً لإبقاء الموقع صاحي (الاستضافات زي Render/Replit/Glitch بتحسب أي طلب HTTP كنشاط)
- حتى لو الصفحة ردّت بتحويل لصفحة تسجيل الدخول (302/redirect) — **السيرفر اتصحّى بالفعل** لأن الطلب وصله
- لو عايز الزيارة تكون **بجلسة مسجلة فعلاً**: سجّل دخول من متصفحك → انسخ الـ Cookie (من DevTools → Application → Cookies) → حطه في حقل `headers` بتاع الرابط:
```json
{ "url": "https://موقعك.com", "headers": { "Cookie": "session=القيمة_هنا" }, "enabled": true }
```
- ⚠️ تسجيل دخول جوجل التفاعلي (OAuth بمتصفح) مش ممكن أتمتته بأمان — جوجل بتمنع البوتات، لكن الـ Cookie بيحل الموضوع

## 🔌 واجهة الـ API
| الطريقة | المسار | الوظيفة |
|---|---|---|
| GET | `/api/links` | كل الروابط + آخر حالة + إحصائيات |
| POST | `/api/links` | إضافة رابط `{url, name?, interval_minutes?, timeout_ms?}` |
| PUT | `/api/links/:id` | تعديل إعدادات رابط |
| DELETE | `/api/links/:id` | حذف رابط وسجلّه |
| GET | `/api/links/:id/checks?limit=30` | سجل الزيارات |
| POST | `/api/links/:id/check` | فحص فوري الآن |
| GET | `/api/tick` | تشغيل الفحوصات المستحقة (نسخة احتياطية للكرون) |
| GET | `/api/summary` | ملخص: الإجمالي / شغال / واقع |

## 🏗️ البنية التقنية (Stack)
- **Backend**: Hono (TypeScript) على Cloudflare Workers — أخف وأسرع إطار للحافة
- **Database**: Cloudflare D1 (SQLite موزعة عالمياً)
- **Scheduler**: Cloudflare Cron Triggers (`* * * * *` — كل دقيقة)
- **Frontend**: HTML + TailwindCSS + Font Awesome + Axios (CDN) — خط Cairo عربي

## 🗄️ نموذج البيانات
- **links**: id, name, url, interval_minutes, method, timeout_ms, headers_json, enabled, last_checked_at
- **checks**: id, link_id, status (up/down), http_status, response_time_ms, error, checked_at
- تنظيف تلقائي: يُحتفظ بآخر 100 فحص لكل رابط فقط

## ⚠️ ملاحظات مهمة
- **الزيارات التلقائية** بتشتغل من سيرفرات Cloudflare — موبايلك مش لازم يكون شغال
- **عرض الموقع (iframe)**: بعض المواقع بتمنع العرض داخل إطار (X-Frame-Options) — ده ميأثرش على الزيارات التلقائية
- **تسجيل دخول جوجل بإيميلات مختلفة لكل رابط**: الزيارات من السيرفر بتحصل بدون جلسات مستخدم (زيارة HTTP عادية) — وهي كافية تماماً لإبقاء الموقع صاحياً. لو محتاج جلسات مسجلة، ممكن تضيف Headers مخصصة (Cookies/Tokens) لكل رابط عبر حقل `headers_json` في الـ API
